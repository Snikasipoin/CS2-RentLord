# CS2 Account Rent Bot

Telegram-бот для управления арендой Steam-аккаунтов под CS2 и Faceit.
Проект построен на Python 3.11, `aiogram 3`, SQLite и `cryptography`.

Бот рассчитан на повседневную работу администратора:
он хранит аккаунты, управляет арендой, показывает статусы, помогает с выдачей данных покупателю и ведёт историю операций.

## Что умеет бот

- Добавлять новые аккаунты и редактировать уже существующие.
- Хранить данные Steam, Faceit и дополнительные служебные поля.
- Сохранять заметки по аккаунту.
- Отмечать еженедельный дроп и автоматически сбрасывать отметку по новой неделе.
- Показывать статус Steam-аккаунта: в игре или нет.
- Показывать, занят аккаунт или свободен, и сколько времени осталось до окончания аренды.
- Продлевать аренду и освобождать аккаунт вручную.
- Вести историю аренд по каждому аккаунту.
- Показывать сводную статистику по арендным операциям.
- Генерировать Steam Guard-коды по `maFile`.
- Генерировать Faceit 2FA-коды по сохранённому секрету.
- Сохранять и применять `maFile` через редактирование карточки аккаунта.
- Поддерживать FunPay-заказы: привязка заказа к аренде, отправка стартовых данных и ручная отправка кодов в чат заказа.
- Поддерживать автоподъём лотов FunPay.
- Хранить чувствительные данные в зашифрованном виде.

## Основные экраны

### Аккаунты

- список всех аккаунтов;
- быстрый переход в карточку аккаунта;
- отображение статуса аренды;
- отображение Steam-статуса;
- отображение блокировок и служебных пометок.

### Карточка аккаунта

- данные Steam и Faceit;
- заметка администратора;
- статус еженедельного дропа;
- статус Steam в игре или вне игры;
- действия по коду Steam и коду Faceit;
- привязка FunPay-заказа;
- история аренды.

### Статус

- общее состояние базы;
- количество свободных и занятых аккаунтов;
- количество аккаунтов с Faceit;
- количество аккаунтов с блокировками;
- статистика аренд;
- служебная информация по интеграциям.

### Данные

- Faceit API key;
- Steam API key;
- FunPay Golden Key;
- FunPay User-Agent;
- резервное копирование и восстановление базы;
- служебные настройки интеграций.

### FunPay

- получение баланса;
- управление автоподъёмом лотов;
- настройка Golden Key и User-Agent;
- привязка заказа к аренде;
- ручная отправка данных и кодов в чат заказа.

## Как устроены данные

- Основная база хранится в `accounts.db`.
- Пароли и секреты шифруются через Fernet.
- Настройки сохраняются в таблице `settings`.
- `maFile` загружается как JSON и используется для Steam Guard и служебных Steam-данных.
- Steam presence-кэш обновляется в фоне и не тормозит открытие карточек.
- FunPay-заказ хранит номер, ссылку, статус, цену, покупателя, чат и время последней синхронизации.

## Важные особенности текущей версии

- Steam Guard-коды генерируются локально из `maFile`.
- FunPay-слушатель чата и запрос кодов от покупателя отключены, чтобы не засорять поток аренды и не создавать нестабильность.
- Коды в FunPay-заказ отправляются вручную через карточку аккаунта.
- Статус завершения FunPay-заказа используется для финального уведомления покупателя при окончании аренды.
- Сообщение за 5 минут до конца аренды просит продлить аренду, а не запрашивать код.

## Установка

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Создайте файл `.env` рядом с `bot.py`:

```env
BOT_TOKEN=your_telegram_bot_token
ADMIN_ID=your_telegram_user_id
ENCRYPTION_KEY=your_fernet_key
```

3. При необходимости добавьте дополнительные переменные:

```env
FACEIT_API_KEY=your_faceit_api_key
STEAM_API_KEY=your_steam_web_api_key
FUNPAY_GOLDEN_KEY=your_funpay_golden_key
FUNPAY_USER_AGENT=your_funpay_user_agent
```

4. Запустите бота:

```bash
python bot.py
```

## Рекомендуемый сценарий работы

1. Добавьте аккаунт в базу.
2. Заполните Steam, Faceit и дополнительные данные.
3. При необходимости загрузите `maFile` через редактирование аккаунта.
4. Настройте API-ключи в разделе `Данные`.
5. Отслеживайте статусы через `Статус` и `Аккаунты`.
6. При сдаче аккаунта укажите срок аренды и, если нужно, FunPay-заказ.
7. Выдавайте код Steam или Faceit из карточки аккаунта одной кнопкой.

## Безопасность

- Не храните `.env` и `accounts.db` в публичном доступе.
- Используйте уникальный `ENCRYPTION_KEY`.
- Не передавайте `maFile` и другие секреты посторонним.
- Для хостинга лучше использовать постоянный каталог данных.
- При резервном копировании выгружайте только зашифрованную базу.
