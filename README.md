# CS2 Account Rent Bot

Telegram-бот для управления арендой Steam-аккаунтов под CS2 и Faceit.
Проект написан на Python 3.11 и использует `aiogram 3`, SQLite, `aiohttp` и `cryptography`.

Бот помогает вести базу аккаунтов, быстро выдавать данные покупателю, отслеживать статусы и не перегружать работу ручными операциями.

---

## Что умеет бот

- добавлять и редактировать аккаунты;
- хранить Steam, Faceit и служебные данные;
- загружать `maFile` прямо в карточку аккаунта;
- генерировать Steam Guard-коды по `maFile`;
- генерировать Faceit 2FA-коды;
- показывать статус Steam-аккаунта в игре или вне игры;
- хранить заметки по аккаунту;
- отмечать еженедельный дроп и автоматически сбрасывать отметку по новой неделе;
- вести историю аренд;
- показывать статистику по арендам;
- привязывать FunPay-заказ к аренде;
- отправлять стартовые данные и коды в чат заказа вручную;
- управлять автоподъёмом лотов FunPay;
- хранить чувствительные данные в зашифрованном виде.

## Карточка аккаунта

В карточке аккаунта отображаются:

- Steam логин и пароль;
- Faceit email и пароль;
- статус аренды;
- Steam статус;
- заметка администратора;
- отметка о еженедельном дропе;
- наличие `maFile`;
- FunPay-заказ и его служебные поля;
- история аренды;
- кнопки для выдачи кодов.

### Действия из карточки

- `Код Steam`;
- `Код Faceit`;
- `Код Steam в заказ`;
- `Код Faceit в заказ`;
- `Редактировать`;
- `Удалить`;
- `Назад`.

## Как устроена логика работы

### Аренда

- при сдаче аккаунта указывается срок аренды;
- бот хранит время окончания;
- за 5 минут до конца аренды отправляется напоминание;
- при окончании аренды бот проверяет Steam-статус;
- если аккаунт в игре, администратор получает предупреждение;
- если аккаунт не в игре, аренда закрывается автоматически.

### FunPay

- заказ можно привязать к аренде по номеру или ссылке вида `https://funpay.com/orders/YAT27MC9/`;
- бот сохраняет номер заказа, ссылку, статус, цену, покупателя и чат заказа;
- стартовые данные отправляются в чат заказа вручную через карточку аккаунта;
- при окончании аренды бот использует статус заказа для финального сообщения покупателю.

### Steam

- `maFile` загружается в аккаунт через редактирование;
- бот извлекает `shared_secret`, `identity_secret`, `device_id`, `SteamID64`, `AccessToken`, `RefreshToken`;
- Steam Guard-код генерируется локально;
- статус Steam-аккаунта берётся из фонового кэша Steam Web API;
- при наличии `Steam API key` бот может показывать, в игре аккаунт или нет.

## Установка

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Создайте `.env` рядом с `bot.py`:

```env
BOT_TOKEN=your_telegram_bot_token
ADMIN_ID=your_telegram_user_id
ENCRYPTION_KEY=your_fernet_key
```

3. Добавьте дополнительные ключи, если они нужны:

```env
FACEIT_API_KEY=your_faceit_api_key
STEAM_API_KEY=your_steam_web_api_key
FUNPAY_GOLDEN_KEY=your_funpay_golden_key
FUNPAY_USER_AGENT=your_funpay_user_agent
```

4. Запустите бота:

```bash
python bot.py
```

## Структура данных

- `accounts.db` — основная SQLite-база.

## Важные особенности текущей версии

- FunPay-слушатель чата и запрос кодов от покупателя отключены.
- Коды в FunPay-заказ отправляются вручную через карточку аккаунта.
- За 5 минут до окончания аренды бот просит продлить аренду.
- После окончания аренды финальное сообщение зависит от статуса заказа.
- Steam presence обновляется в фоне и не блокирует открытие интерфейса.
- `maFile` можно загрузить через окно редактирования аккаунта.

## FAQ

### Можно ли выдавать Steam и Faceit коды отдельно?

Да. В карточке аккаунта есть отдельные кнопки для Steam и Faceit.

### Можно ли автоматически показывать, в игре аккаунт или нет?

Да, если указан `Steam API key` и у аккаунта есть `SteamID64`.

### Нужен ли `maFile` для Steam Guard-кодов?

Да, `maFile` нужен для локальной генерации Steam Guard-кода.

### Почему FunPay-команды от покупателя отключены?

Чтобы не засорять основной поток аренды и не зависеть от нестабильной логики чатов FunPay.

### Можно ли вернуть отправку кодов в чат заказа позже?

Да, архитектура под это сохранена, но сейчас рабочий сценарий сделан через ручную выдачу из карточки аккаунта.

## Безопасность

- не публикуйте `.env` и `accounts.db`;
- используйте уникальный `ENCRYPTION_KEY`;
- не передавайте `maFile` посторонним;
- храните резервные копии в защищённом месте;
- по возможности ограничьте доступ к боту только доверенными администраторами.
